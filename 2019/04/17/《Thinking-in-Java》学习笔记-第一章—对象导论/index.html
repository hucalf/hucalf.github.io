笔者前言

最近在阅读《Thinking in Java》，借助这本书呢，我这这里认真梳理Java基础知识，为了形成自己的知识体系。坚持把这一系列的学习笔记。一是为了提升自己的学习能力；二是给自己整理一份资料，方便以后的查阅和复习。同时也希望能够帮助更多的小伙伴:

面向对象程序设计（Object—oriented Programming，OOP）

抽象过程

Java的五个基本特性,表现了一种存粹面向对象程序设计方式:
1) 万物皆对象。
2) 程序是对象的集合。
3) 每个对象都有自己的由其他对象所构成的存储。
4) 每个对象拥有其类型。
5) 某一特定类型所有对象都已接收同样的消息。

我们可以简洁的描述为:对象具有状态、行为、标识。

每个对象都有一个接口

注:这里的接口（interface）的定义，并不是我们认为的interface,而是每个对象对外提供的方法。

接口定义了某一特定的对象所能发出的请求。就是向某个对象”发送请求”，这个对象便知道此消息的目的，然后执行对应的程序的代码。

为了向对象发送消息，需要声明对象的名称，并以圆点符号连接一个请求。

每个对象都提供服务

将对象看作是服务提供者:它有助于提高对象的内聚性。它是一个简化工具，不仅在设计过程中非常有用，而且当其他人试图理解你的代码或重用某个对象时，如果他们看出对象所能提供的服务的价值，它会使调整对象以适应其设计的过程变得简单得多。

被隐藏的具体实现

程序开发人员分为类创建者和客户端程序员。

  • 类创建者目标是构建类:只向客户端程序员暴露必须的部分，而隐藏其他部分。被隐藏的部分目的是为了不被客户端程序员所毁坏，这样可以减少程序bug。

  • 客户端程序员:每当类创建者创建一个类库时，就建立了与客户端程序员之间的关系，使用这个类库来构建客户端应用。

访问控制是为了能够让客户端程序员只接触自己应该所触及的部分；还有就是允许库设计者可以改变类内部的工作方式而不用担心会影响到客户端程序员。

复用具体实现

代码复用是面向对象程序设计语言所提供的最了不起的优点之一。组合”has-a(拥有)”关系（composition）就是使用现有的类合成新的类。如果组合是动态发生的，便称之为聚合（aggregation）。

特点:极大的灵活性。在我们创建新类时，首先考虑组合，这样会使设计变得更加清晰。

继承

继承(extends)使用基类型和导出类型的概念表示了这种类型之间的相似性。一个基类包含其所有导出类型共享的特性和行为。何以创建一个基类型来表示系统某些对象的核心概念，从基类型中导出其他类型，来表示此核心可以被实现的各种不同方式。

当继承现有类时，也就创造了新的类型。意味着导出类与基类具有相同的类型。

基类与导出类产生差异的两种方法:

1) 当不能满足你所有的需求，你可以直接在导出类添加新的方法。
2) 覆盖（overriding）就是改变现有基类的方法的行为。

“是一个” 与 “像是一个” 关系

is-a(是一个):替代原则:继承应该只覆盖基类的方法，意味着导出类和基类是完全相同的类型，因此它们具有完全相同的接口，可以用一个导出类对象完全替代一个基类对象。
确定是否用is-a来描述基类与导出来之间的关系，来判断是否继承。

is-like-(像是一个):有时必须在导出类型中添加新的接口元素，这样扩展了接口，这个新的类型可以代替基类。（基类无法访问新添加的方法）

伴随多态的可互换性

多态使得事情总能够正确的处理。当向一个对象发送消息时，即涉及向上转型，该对象也知道要执行什么样的行为。

前期绑定:一个非面向对象编程的编译器产生的函数调用。

后期绑定:当向对象发送消息时，被调用的代码直到运行时才能确定。

单根继承结构

单根继承结构的优点:

  • 在单继承结构中的所有的对象都具有一个共用的接口（相同的基本类型）。
  • 单继承结构保证所有对象都具备某些功能
  • 单继承结构使垃圾回收器的实现变得容易得多，编程带来更大的灵活性。

容器

容器也被称之为集合。

对容器的选择有两个原因:

1) 不同容器提供了不同的接口和外部行为，堆栈相比于队列就具备不同的接口和行为，也不同于集合和列表的接口行为。它们之中的某种容器提供的解决方案可能比其他容器要灵活得多。
2) 不同的容器对于某些操作具有不同的效率。

ArrayList和LinkedList的比较:它们都具有相同的接口和外部行为的简单的序列，不同的是：在ArrayList中，随机访问元素是一个花费固定时间的操作，在LinkedList中，随机选取元素在列表移动，这种代价是高昂的，访问元素越接近表尾的元素，花费的时间越长。如果想要在序列中间插入一个元素，LinkedList的开销却比ArrayList要小。

性能优化选择ArrayList。

对象的创建和生命期

在使用对象时，最关键的问题之一便是它们的生成和销毁方式。每个对象为了生存都需要资源，尤其时内存，当我们不需要一个对象时，它必须被清理，使其占有的资源可以被释放和重用。

Java提供了被称为”垃圾回收器”的机制，它可以自动发现对象何时不再被使用，并继而销毁他。用来处理内存的释放。

异常处理:处理错误

异常提供了一种错误状况进行可靠恢复的途径。如果代码没有编写正确，你就会得到编译时的出错消息，你可以进行改正，并恢复你执行的程序，有助于编写处更健壮的代码。

并发进程

并发是指有时中断处理时间性强的任务是必须的，但是对于大量的其他问题，我们只想把问题切分成对个可独立运行的部分（线程），从而提高程序的响应能力。

并发它有一个隐患：共享资源。如果有多个并行任务都要访问同一资源，就会出现问题。为了解决这个问题，某个任务锁定某项资源，完成其任务，然后释放资源锁，使其他任务可以使用这项资源。

Java与Internet

Web是什么

1) 客户/服务器计算技术

客户/服务器系统的核心技术思想是：系统具有一个中央信息存储池（central repository of information），用来存储某种数据，它通常存在于数据库中，你可以根据需要将它分发给某些人员或机器集群。

2) Web就是一台举行服务器

所有的服务器和客户机都存在于同一个网络。

客户端编程

Web最初的”服务器—浏览器”设计是为了能够提供交互性的内容。基本的HTML（Hyper Text Markup Language，超文本标记语言）。

服务端编程

Java中的Web服务器，它让你使用Java编写被称之为servlet的程序来实现服务器编程。servlet及其衍生物JSP。

总结

面向对象编程，编写良好的Java程序，易于理解。对于设计良好的程序，通常阅读它就可以很容易地理解其代码。
